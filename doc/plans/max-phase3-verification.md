# План ручной проверки: Phase 3 MAX (карточка сотрудника, договоры, привязка мессенджеров)

## 1. Карточка сотрудника (TG + MAX)

**Файлы:** `owner.py`, `detail.html`

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1.1 | Войти как owner → Сотрудники → открыть карточку сотрудника с TG | Видны бейджи: TG (с ID), MAX (если привязан) |
| 1.2 | Открыть карточку сотрудника только с MAX, без TG | Бейдж MAX отображается |
| 1.3 | Открыть карточку сотрудника без привязок | Нет бейджей или «не привязаны» |
| 1.4 | Клик «Новый договор» из карточки | Переход на форму создания с предзаполненным `employee_id` |

## 2. Договоры без обязательного TG

**Файлы:** `owner.py`, `contract_service.py`, `create.html`

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 2.1 | Сотрудники → Новый договор (без employee_id в URL) | Селект сотрудников + поле Telegram ID (не required) |
| 2.2 | Выбрать сотрудника из селекта, заполнить форму, создать | Договор создаётся по employee_id |
| 2.3 | Ввести только Telegram ID (сотрудник не в списке), создать | Создаётся User при отсутствии, договор привязывается |
| 2.4 | Оставить оба поля пустыми, Submit | Ошибка: «Укажите сотрудника» |
| 2.5 | Из карточки сотрудника клик «Новый договор» | Форма с hidden employee_id, сотрудник уже выбран |
